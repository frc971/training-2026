# Lesson 07-08 Task: Real Contribution Transition

# IMPORTANT: DO NOT ATTEMPT THIS TASK WITHOUT READING [LESSON 07-08](../../lesson-07-08-transition-to-software-member/README.md)

In this task, you'll transition from training exercises into a real scoped contribution.

## Your Mission

Take one real mentor-approved task from:

- 3rd robot work
- an existing 2026 robot repo
- a scoped sim/logging/helper task

The task should be small enough to finish with support but real enough that it matters.

## Before Starting

Write down or explain:

- what the task is
- why it matters
- what files you expect to touch
- how you plan to test it
- when you will need review or robot help

## Expectations

- Make progress independently before asking for rescue.
- Read surrounding code before editing.
- Do not treat GPT as the implementation source of truth.
- If robot testing is needed, pair with an experienced member.
- Be ready to respond to review comments quickly.

## Successful Completion Looks Like

- the task is finished or moved forward substantially
- you can explain what you changed
- you can explain what evidence you have
- you can explain what still needs help, if anything

## Why This Matters

- Training only matters if it turns into real contribution.
- The goal is not to make you solo every robot problem immediately.
- The goal is to make you useful, reliable, and capable of making progress with support.

## Wrap-Up

Once the task reaches a reasonable stopping point:

1. Review the outcome with a mentor.
2. Summarize what you learned.
3. Identify the next thing you still need practice on:
   - code reading
   - Java features
   - debugging
   - sim
   - deploy
   - on-robot testing
